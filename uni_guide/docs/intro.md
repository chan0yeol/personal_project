# 실전 가이드 ​

UniWORKS 개발에 필요한 핵심 기능을 주제별로 정리했습니다

## 개요 ​

UniWORKS 개발을 시작하시나요?
 이 가이드는 실무에서 가장 많이 사용하는 기능들을 예제와 함께 설명합니다.

## 주요 모듈 ​

### Grid ​

UniWORKS의 핵심 테이블 UI 컴포넌트입니다. 데이터를 표 형태로 보여주고 편집할 수 있습니다.

| 기능 | 설명 |
| 데이터 접근 | 셀 데이터 읽기/쓰기/찾기 (`$V`, `$F`) |
| 데이터 조회 | 전체 또는 선택된 행 데이터 조회 |
| 행 조작 | 행 추가, 삭제, 이동 |
| 셀 제어 | 편집 가능 여부, 필수 입력 설정 |
| 컬럼 제어 | 컬럼 숨김, 편집 가능 여부 설정 |
| 이벤트 처리 | 클릭, 값 변경 이벤트 핸들링 |
| 검증 | 데이터 유효성 검사 |
👉 Grid 가이드 바로가기

### 유틸리티 ($u) ​

UniWORKS 개발의 기본이 되는 핵심 유틸리티 모듈입니다.
 폼 제어, 페이지 이동, 서버 통신 등 일상적인 개발 작업을 간편하게 처리할 수 있습니다.

| 기능 | 설명 |
| 값 접근 | 폼 필드 읽기/쓰기 |
| 페이지 제어 | 페이지 이동, 새로고침 |
| 다이얼로그 | 팝업, 알림, 입력창 표시 |
| Ajax 통신 | 서버와 데이터 통신 |
| 버튼 핸들러 | 버튼 클릭 이벤트 처리 |

## 캐시 (Cache) ​

브라우저 캐시와 서버 캐시를 이해하면 "왜 내 수정이 반영 안 되지?" 같은 문제를 쉽게 해결할 수 있습니다.

### 캐시 종류 ​

| 종류 | 내용 | 예시 |
| 브라우저 캐시 | JS, CSS, 이미지 등 정적 파일을 로컬에 저장 | 자바스크립트 파일 수정 |
| 서버 캐시 | WEB_DATA, SAP RFC 메타데이터를 서버 메모리에 저장 | 화면 설정 변경 |
| 캐시 버스팅 | requireBust, webDataCacheBust로 캐시 갱신 제어 | 배포 후 강제 갱신 |
### 자주 만나는 캐시 문제 ​

| 증상 | 원인 | 해결방법 |
| "JS 파일 수정했는데 반영 안돼요" | 브라우저 캐시 | `Ctrl + F5`로 강력 새로고침 |
| "WEB_DATA 수정했는데 안 바뀌어요" | 서버 캐시 | 일반 새로고침 (자동 갱신) |
| "SAP 테이블 구조가 인식 안돼요" | 서버 메타데이터 캐시 | Debug 화면에서 캐시 클리어 |
