# 코딩 가이드 ​

더 나은 코드를 작성하기 위한 실무 노하우와 권장 패턴

## 이 가이드를 만든 이유 ​

"이 코드 왜 이렇게 짰어?" 하는 비판이 아니라, "이렇게 하면 더 좋아질 것 같아요!" 하는 제안입니다. 실무에서 겪은 시행착오를 바탕으로, 코드를 더 읽기 쉽고 유지보수하기 좋게 만드는 방법을 정리했습니다.

## 주요 주제 ​

### 💎 코드 품질 ​

코드를 더 안전하고 읽기 쉽게 만드는 방법입니다.

| 가이드 | 이런 내용이에요 | 배울 수 있어요 |
| const/let 활용 | var 대신 const와 let을 쓰면 왜 좋을까요? | 블록 스코프, 안전한 변수 관리 |
| 네이밍 규칙 | 변수/함수 이름을 명확하게 짓는 법 | 의도가 드러나는 코드 작성 |
| 매직 넘버 제거 | 숫자 100이 뭘 의미하는지 알 수 있게 | 상수 활용으로 가독성 향상 |
### 🎯 ES6+ 문법 ​

최신 JavaScript 문법으로 코드를 더 간결하게 작성하는 방법입니다.

| 가이드 | 이런 내용이에요 | 배울 수 있어요 |
| ES6+ 활용 | 화살표 함수, 구조 분해 등 현대 문법 | 간결하고 표현력 있는 코드 |
| 반복문 성능 | for, forEach, map 중 뭘 써야 할까? | 상황별 최적의 반복 방법 |
### ⚡ 비동기 프로그래밍 ​

UniWORKS의 서버 통신을 더 깔끔하게 처리하는 방법입니다.

| 가이드 | 이런 내용이에요 | 배울 수 있어요 |
| 동기 vs 비동기 | 비동기가 뭔지, 왜 필요한지 기초부터 | 비동기 개념 이해 |
| Promise/async-await | 콜백 지옥을 벗어나는 현대적인 방법 | 깔끔한 비동기 코드 작성 |
| 콜백 지옥 해결 | 중첩된 콜백을 리팩토링하는 실전 예제 | 가독성 높은 비동기 처리 |
### 🔧 함수형 프로그래밍 ​

반복문 대신 map, filter를 쓰면 코드가 더 읽기 쉬워집니다.

| 가이드 | 이런 내용이에요 | 배울 수 있어요 |
| 함수형 기초 | map, filter, reduce로 데이터 다루기 | 선언적이고 명확한 코드 작성 |
## 빠른 맛보기 ​

Before와 After를 비교해보세요. 어떤 코드가 더 읽기 쉬운가요?

javascript```
// ❌ Before - 반복문으로 필터링
var activeData = [];
$.each(gridData, function(i, item) {
    if (item.STATUS === 'A') {
        activeData.push(item);
    }
});

// ✅ After - filter로 간결하게
const activeData = gridData.filter(item => item.STATUS === 'A');```
두 코드는 같은 일을 하지만, 아래 코드가 훨씬 의도가 명확하죠?

## 우선순위 가이드 ​

코드를 작성할 때 무엇을 우선해야 할까요?

| 상황 | 우선순위 | 이유 |
| 일반 개발 | 가독성 > 성능 | 나중에 다른 사람(또는 미래의 나)이 이해하기 쉬워야 해요 |
| 대용량 데이터 처리 | 성능 > 가독성 | 사용자 경험이 우선이에요 |
| 레거시 코드 수정 | 명확성 > 간결성 | 버그 없이 안전하게 수정하는 게 중요해요 |
