# Slack 알림 봇 - 기능 및 로직 정리

## 개요

유니포스트 사내 시스템(휴가/회의실/출퇴근)의 데이터를 주기적으로 수집하여 Slack 채널 및 DM으로 알림을 보내는 Node.js 봇.

---

## 1. 휴가 알림

### 1-1. 이번주 휴가 (vacationWeekly)

| 항목 | 내용 |
|------|------|
| 스케줄 | 매주 **월요일 09:00** (`0 9 * * 1`) |
| 대상 | 팀 설정에서 `vacationWeekly` 활성화된 웹훅 채널 |
| 동작 | 이번주 월~금 휴가자를 팀별 필터링하여 발송 |

**로직 흐름:**
1. 이번주 월~금 날짜 배열 생성
2. AVS API에서 해당 기간 휴가 데이터 조회
3. 휴가알림 OFF 멤버 제외 (`member.notify.vacation === false`)
4. `dedupeByWebhook("vacationWeekly")`로 같은 웹훅을 공유하는 팀 병합
5. 각 웹훅별로 해당 팀(`deptName`) 소속 휴가자만 필터링하여 발송

**로그:**
```
[이번주휴가] 조회 완료 (2026-04-07~2026-04-11, 전체 5명)
  → 채널(구독2팀,구독4팀,구독6팀) 발송: 3명 | 홍길동(2026-04-08), ...
```

### 1-2. 오늘 휴가 (vacationDaily)

| 항목 | 내용 |
|------|------|
| 스케줄 | **화~금 09:00** (`0 9 * * 2-5`) |
| 대상 | 팀 설정에서 `vacationDaily` 활성화된 웹훅 채널 |
| 동작 | 오늘 휴가자를 팀별 필터링하여 발송 |

**로그:**
```
[오늘휴가] 조회 완료 (2026-04-08, 전체 3명)
  → 채널(구독2팀,구독4팀,구독6팀) 발송: 2명 | 홍길동(연차), 김철수(반차)
```

### 1-3. 휴가 변동 감지 (10분 동기화)

| 항목 | 내용 |
|------|------|
| 스케줄 | **10분마다** (`*/10 * * * *`) |
| 대상 | 팀 설정에서 `vacationChange` 활성화된 웹훅 채널 |
| 동작 | 이전 스냅샷과 비교하여 추가/취소된 휴가를 실시간 알림 |

**로직 흐름:**
1. 오늘~60일 후 범위로 휴가 데이터 조회
2. 이전 스냅샷(`prevVacationSnapshot.map`)과 현재 데이터 비교
3. **추가 감지**: 현재 있는데 스냅샷에 없는 항목
4. **취소 감지**: 스냅샷에 있는데 현재 없는 항목 중 `useEdate >= today` (날짜 경과 오작동 방지)
5. 휴가알림 OFF 멤버 제외
6. `dedupeByWebhook("vacationChange")`로 팀별 필터링 후 발송
7. 스냅샷 갱신 및 저장

**로그:**
```
[변동 감지] 휴가 변경 사항 발견 (추가: 2, 취소: 1)
  + [휴가추가] 홍길동(구독2팀) 2026-04-09~2026-04-10 연차
  + [휴가추가] 김철수(구독4팀) 2026-04-08 반차 오전 (09:00~13:00)
  - [휴가취소] 이영희(구독6팀) 2026-04-08 연차
  → 웹훅 발송 (구독2팀,구독4팀,구독6팀): 추가 2건, 취소 1건
```

### 멤버별 휴가 알림 제외

- `member.notify.vacation === false`이면 모든 휴가 알림에서 해당 인원 제외
- Web UI 멤버 탭 "🏖 휴가알림" 배지로 토글

## 2. 회의실 알림

### 2-1. 오늘 회의실 (roomDaily)

| 항목 | 내용 |
|------|------|
| 스케줄 | **월~금 09:00** (`0 9 * * 1-5`) |
| 대상 | 팀 설정에서 `roomDaily` 활성화된 웹훅 채널 |
| 동작 | 오늘 회의실 예약 목록 발송 |

**로그:**
```
[오늘회의실] 조회 완료 (2026-04-08, 2건)
  → 채널(구독2팀,구독4팀,구독6팀) 발송: 2건 | 중회의실(에땅) 10:00~11:00, ...
```

### 2-2. 회의실 변동 감지 (10분 동기화)

| 항목 | 내용 |
|------|------|
| 스케줄 | **10분마다** (sync 크론과 동시) |
| 대상 | 팀 설정에서 `roomChange` 활성화된 웹훅 채널 |
| 동작 | 이전 스냅샷과 비교하여 추가/취소된 회의실 예약을 실시간 알림 |

**로직 흐름:**
1. 오늘 회의실 데이터 조회
2. 이전 스냅샷 날짜가 오늘과 일치할 때만 비교 (날짜 불일치 시 스킵)
3. 추가/취소 감지 후 활성화된 웹훅으로 발송
4. 스냅샷 갱신

**로그:**
```
[변동 감지] 회의실 변경 사항 발견 (추가: 1, 취소: 0)
  + [회의실추가] 중회의실(에땅) | "주간 스프린트" | 홍길동 | 2026-04-08 10:00~11:00
  → 웹훅 발송 (구독2팀,구독4팀,구독6팀)
```

## 3. 출근 미등록 알림

### 3-1. 출근 미등록 DM (commute)

| 항목 | 내용 |
|------|------|
| 스케줄 | **월~금, 08~16시, 매시 55~59분** (`* 8-16 * * 1-5`, 코드에서 55분 미만 스킵) |
| 대상 | 등록된 멤버 중 미출근자에게 **Slack DM** |
| 동작 | 예정 출근시간 5분 전부터, 미등록자에게 1일 1회 DM 발송 |

**로직 흐름:**
1. 날짜 변경 시 `commuteNotifiedToday` Set 초기화
2. 멤버에 등록된 deptId로 출퇴근 API 조회 (팀 활성화 여부 무관)
3. 각 미출근자에 대해:
   - `workStartTime` 있으면 이미 출근 → 스킵
   - 오늘 이미 알림 보냈으면 → 스킵
   - 예정 출근시간(`workStime`) - 5분 이전이면 → 스킵
   - `membersData`에서 매칭 멤버 찾기 (이름 + deptId)
   - `notify.commute === false`이면 → 스킵
   - 조건 통과 시 DM 발송 + Set에 추가 (하루 1회 보장)

**로그:**
```
[출근알림] 체크 대상 deptIds: 0009 (멤버 4명)
[출근알림] DM 발송: 홍길동 (예정: 09:00, 현재: 08:55)
[출근알림] 김철수 - 출근알림 OFF
```

### 3-2. 월말 출근 미등록 리포트 (monthlyCommute)

| 항목 | 내용 |
|------|------|
| 스케줄 | **매주 월요일 09:00** (`0 9 * * 1`), 해당 월 마지막 월요일에만 실행 |
| 대상 | `isLeader: true` 멤버에게 **Slack DM** |
| 동작 | 해당 월 팀원별 출근 미등록 날짜 리포트 발송 |

**로직 흐름:**
1. `isLastMondayOfMonth()` 체크 → 아니면 스킵
2. 해당 월 전체 휴가 데이터 조회 → 사람별 휴가 날짜 Set 생성
3. 팀장별 팀원 수집 → 출퇴근 데이터 조회
4. 미등록 판정 (3가지 모두 충족):
   - 출퇴근 API 목록에 있는 날 (없는 날 = 무시)
   - `workStartTime` 없음 (미등록)
   - 주말 아님 (토/일 제외)
   - 해당 날짜에 휴가 아님 (휴가 데이터와 교차 비교)

**리포트 형식:**
```
📊 구독6팀 2026년 04월 출근 미등록 현황

오찬열: 미등록 없음

홍길동: 미등록 2일
    - 2026-04-03
    - 2026-04-15

기준일: 2026-04-01 ~ 2026-04-28
```

**로그:**
```
[월말출근리포트] 구독6팀 팀장(오찬열)(U09SJ2F3TL1) DM 발송 | 오찬열:0일, 홍길동:2일
```

## 4. Slack 슬래시 커맨드

### `/휴가`

| 입력 | 결과 |
|------|------|
| `/휴가` | 오늘 휴가자 (요청자 팀 그룹 자동 필터) |
| `/휴가 오늘` | 오늘 휴가자 |
| `/휴가 내일` | 내일 휴가자 |
| `/휴가 이번주` | 이번주 (월~금) 휴가자 |
| `/휴가 다음주` | 다음주 (월~금) 휴가자 |
| `/휴가 2026-04-10` | 특정일 휴가자 |
| `/휴가 2026-04-07~2026-04-11` | 기간 휴가자 |
| `/휴가 246` | 구독2·4·6팀 오늘 휴가자 |
| `/휴가 135` | 구독1·3·5팀 오늘 휴가자 |
| `/휴가 1팀 3팀` | 구독1팀·구독3팀 오늘 휴가자 |
| `/휴가 이번주 246` | 구독2·4·6팀 이번주 휴가자 (날짜 + 팀 조합 가능) |
| `/휴가 업데이트` | 최신 데이터 강제 동기화 |

**팀 자동 감지 우선순위:**
1. 팀명/그룹 직접 입력 (`246`, `135`, `1팀` 등) → 해당 팀만 표시
2. 요청자 Slack 프로필에서 팀명 추출 → 같은 그룹 표시
3. 커맨드 입력 채널 ID 매칭 → 해당 채널 팀 표시
4. 모두 실패 → 전체 표시

**날짜 입력 지원 형식:**
```
2026-04-10 / 2026.04.10 / 2026/04/10 / [2026-04-10] / 20260410
```

### `/회의실`

| 입력 | 결과 |
|------|------|
| `/회의실` | 오늘 회의실 예약 현황 |
| `/회의실 오늘` | 오늘 회의실 예약 |
| `/회의실 내일` | 내일 회의실 예약 |
| `/회의실 이번주` | 이번주 회의실 예약 |
| `/회의실 다음주` | 다음주 회의실 예약 |
| `/회의실 2026-04-10` | 특정일 회의실 예약 |
| `/회의실 2026-04-07~2026-04-11` | 기간 회의실 예약 |
| `/회의실 업데이트` | 최신 데이터 강제 동기화 |

## 5. 팀/채널 필터링 구조

### teams.json 구조

```json
{
  "teams": ["구독2팀", "구독4팀", "구독6팀"],
  "webhook": "https://hooks.slack.com/...",
  "channel_id": "C0ANGG70WBD",
  "deptId": "0009",
  "notify": {
    "vacationChange": true,
    "vacationWeekly": true,
    "vacationDaily": true,
    "roomChange": true,
    "roomDaily": true,
    "commute": true
  }
}
```

- `teams`: 이 웹훅으로 알림 받을 팀 목록 (유니포스트 부서명 기준)
- `deptId`: 유니포스트 출퇴근 API 조회용 부서 ID
- `channel_id`: `/휴가` 커맨드 채널 자동 감지용
- `notify`: 알림 유형별 ON/OFF (미설정 시 `enabled` 필드로 폴백)

### dedupeByWebhook 패턴

같은 웹훅 URL을 공유하는 팀 설정이 여러 개일 때, 병합하여 한 번만 발송.

## 6. 멤버별 개별 설정

### members.json 구조

```json
{
  "deptId": "0009",
  "teamName": "구독6팀",
  "name": "오찬열",
  "slackId": "U09SJ2F3TL1",
  "isLeader": false,
  "notify": {
    "commute": true,
    "vacation": true
  }
}
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `isLeader` | `false` | 팀장 여부 (월말 리포트 DM 수신) |
| `notify.commute` | `true` | 출근 미등록 DM 수신 여부 |
| `notify.vacation` | `true` | 휴가 알림에 이 인원 포함 여부 |

Web UI 멤버 탭에서 토글: ★팀장 / 🏢출근알림 / 🏖휴가알림

## 7. 크론 스케줄 요약

| ID | 설명 | 기본 스케줄 | 발송 대상 |
|----|------|-------------|-----------|
| `sync` | 10분 동기화 (휴가+회의실 변동 감지) | `*/10 * * * *` | 채널 웹훅 |
| `vacationWeekly` | 이번주 휴가자 | `0 9 * * 1` (월) | 채널 웹훅 |
| `vacationDaily` | 오늘 휴가자 | `0 9 * * 2-5` (화~금) | 채널 웹훅 |
| `roomDaily` | 오늘 회의실 | `0 9 * * 1-5` (월~금) | 채널 웹훅 |
| `commute` | 출근 미등록 DM | `* 8-16 * * 1-5` (매시 55분 이후만 실행) | 개인 DM |
| `monthlyCommute` | 월말 출근 미등록 리포트 | `0 9 * * 1` (마지막 월요일만) | 팀장 DM |
| `sessionKeep` | 세션 유지 | `*/30 * * * *` | - |
| `logReset` | 로그 초기화 | `0 0 * * 1` (월) | - |

> 크론 스케줄은 Web UI → 크론 탭에서 수정 가능. 변경 시 `settings.json`에 저장되어 재시작 후에도 유지.

## 8. 알림 테스트 (Web UI)

설정 탭 → "알림 테스트" 카드에서 수신할 Slack User ID 입력 후 버튼 클릭.

| 버튼 | 동작 |
|------|------|
| 이번주 휴가 | 실제 API 데이터로 이번주 휴가자 DM 발송 |
| 오늘 휴가 | 실제 API 데이터로 오늘 휴가자 DM 발송 |
| 휴가 변동 | 샘플 데이터로 변동 알림 DM 발송 |
| 오늘 회의실 | 실제 API 데이터로 오늘 회의실 DM 발송 |
| 회의실 변동 | 샘플 데이터로 변동 알림 DM 발송 |
| 출근 미등록 | 샘플 메시지 DM 발송 |
| 월말 출근 리포트 | 실제 API 데이터로 이번달 미등록 현황 DM 발송 |

- 구독1~7팀 데이터만 필터링하여 표시
- 입력한 Slack User ID는 localStorage에 저장되어 유지
