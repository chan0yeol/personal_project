## 설치 방법

1. Chrome 브라우저 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 토글 ON
3. **압축해제된 확장 프로그램 로드** 클릭
4. 아래 경로 선택

```
\\dockersys.synology.me\maintenance\team6\common\UniSupport-extension
```

> 네트워크 드라이브가 연결되어 있어야 합니다.

### 업데이트 적용

파일이 변경된 경우 `chrome://extensions` 에서 익스텐션 카드의 **새로고침(↺) 버튼** 클릭 후, 사이트 탭도 **F5** 로 새로고침해야 합니다.

> 탭 새로고침을 안 하면 구 버전 스크립트가 계속 실행됩니다.

## 백엔드 서버 연결

반영일정 등록·조회·알림 기능은 내부 백엔드 서버(`192.168.10.54:3001`)를 사용합니다.  
서버가 실행 중이지 않으면 반영일정 관련 기능이 동작하지 않습니다.

| 항목 | 값 |
|------|-----|
| 서버 주소 | `http://192.168.10.54:3001` |
| 실행 경로 | `D:\999.오찬열\00.project\slack\unisupport-server` |
| 실행 방법 | `node server.js` 또는 `npm start` |
| DB | PostgreSQL — `deploy_schedules` 테이블 |

> `new-deploy` 서버의 반영일정 크론은 비활성화되어 있습니다. `unisupport-server` 하나만 실행해야 중복 알림이 발생하지 않습니다.

AI 유사사례 검색·답변 초안 생성 기능은 별도의 AI 서버(`192.168.10.54:8000`)를 사용합니다.

| 항목 | 값 |
|------|-----|
| 서버 주소 | `http://192.168.10.54:8000` |
| 실행 경로 | `D:\999.오찬열\00.AI\01.Support\support-ai\mcp_server` (`api_server.py`) |
| 실행 방법 | `docker compose up -d --build api_server` |
| 검색 방식 | BGE-M3 dense 단일 검색 (threshold 0.70) |
| 컬렉션 | `support_cases_hyde`(문의) / `support_answers_hybrid`(처리내역) / `support_full`(전체유형) |

## FAB 버튼

`home.uni` / `welcome.uni` 페이지 접속 시 화면 **우측 하단**에 파란색 원형 버튼(FAB)이 나타납니다.

- **클릭** → 메뉴 펼치기 / 닫기
- **드래그** → 위치 이동 가능
- **ESC** → 열려있는 메뉴 또는 패널 닫기
- 메뉴 아이템에 **마우스 오버** → 해당 기능 간략 설명 툴팁 표시

## 핀 고정 버튼

각 메뉴 아이템 우측의 **핀 아이콘**을 누르면 해당 기능을 화면 어디서든 바로 사용할 수 있는 독립 플로팅 버튼으로 고정할 수 있습니다.

- **핀 아이콘 클릭** → 플로팅 버튼 생성 (파란색으로 활성화됨)
- **다시 클릭** → 플로팅 버튼 제거
- **플로팅 버튼 드래그** → 원하는 위치에 배치
- 위치는 **localStorage**에 저장되어 새로고침 후에도 유지됨
- **F8** → 플로팅 버튼의 삭제(×) 버튼 표시/숨김 토글

## 기능 상세

### 반영일정 조회

등록된 반영일정을 조회합니다.

- **담당자** 이름 또는 **고객사** 이름으로 필터링 (오토컴플리트 지원)
- 결과 목록: 접수번호 · 고객사명 · 내용 · 반영일시 · 남은시간
- **접수번호 클릭** → 해당 티켓 화면으로 이동
- 남은시간 색상 표시

  | 색상 | 조건 |
  |------|------|
  | 회색 | 완료 (반영시각 경과) |
  | 기본 | 24시간 이상 남음 |
  | 주황 | 1시간 ~ 24시간 |
  | 빨강 | 1시간 미만 |

- 본인이 등록한 일정은 **수정 / 삭제** 가능
- 수정 시 알림 시간도 변경 가능

### 반영일정 등록

팀·담당자를 선택하고 반영 일정을 등록합니다.

| 항목 | 설명 |
|------|------|
| 팀 | 드롭다운 선택 (Slack 구독팀 기준 자동 로드) |
| 담당자 | 팀 선택 후 해당 팀원 목록 |
| 반영일시 | 날짜 + 시간 입력 |
| 고객사 | 현재 열려있는 티켓의 고객사명 자동 입력 (수정 가능) |
| 접수번호 | 현재 열려있는 티켓 번호 자동 입력 (수정 가능) |
| 제목 | 반영 내용 직접 입력 |
| 알림 시간 | 반영 몇 분 전 알림 수신 여부 **다중 선택** |

#### 알림 시간 선택

등록 시 **5분 / 10분 / 15분 / 30분** 중 원하는 시간을 복수 선택할 수 있습니다.  
선택한 각 시간마다 담당자 Slack DM이 발송됩니다.

> ※ 본 기능은 업무 편의를 위한 보조 알림 기능입니다.  
> 알림 누락·지연 가능성이 있으므로 중요 일정 및 만료 여부는 반드시 별도 확인 바랍니다.

- 반영일정 조회 목록의 접수번호 클릭과 동일한 방식으로 동작

### 검색 설정

조회 조건을 설정하고 저장합니다.

#### 처리자
- 이름 직접 입력
- 저장값 없으면 현재 로그인 계정 이름 자동 입력

#### 시작일 (단일 선택)

| 옵션 | 기간 |
|------|------|
| 10일 전 | 오늘 기준 10일 전 **(기본값)** |
| 한달 | 30일 전 |
| 3개월 | 90일 전 |
| 6개월 | 180일 전 |
| 360일 | 360일 전 |

> 종료일은 항상 **오늘 날짜**로 자동 설정됩니다.

#### 팀 (다중 선택)
- 1팀 ~ 7팀, 미지정, 전체 선택 가능

#### 처리상태 (다중 선택)

| 코드 | 상태 |
|------|------|
| R | 확인요청 |
| E | 테스트요청 |
| O | 운영반영요청 |
| A | 접수 |
| W | 보류 |
| C | 처리중 |
| M | 검토 |
| N | 고객사답변 |

> 기본값: 보류(W) 제외 전체

#### 정렬 기준
- **기준**: 처리상태 / 고객사명 / 처리일
- **방향**: 오름차순 / 내림차순

#### 화면 진입 시 자동 설정
- **ON** : 페이지 접속마다 저장된 조건으로 자동 조회 실행
- **OFF** : 버튼 클릭 시에만 조회

#### 버튼

| 버튼 | 동작 |
|------|------|
| 닫기 | 저장 없이 패널 닫기 |
| 저장 | 설정만 저장 (조회 실행 안 함) |
| 저장 & 조회 | 설정 저장 후 즉시 조회 실행 |

### AI 유사사례 검색

현재 문의(접수 상세 페이지)와 유사한 과거 처리 사례를 AI로 검색합니다.

1. **문의 내용**: 패널을 열면 현재 페이지의 `REQ_TITLE`(제목) + `REQ_TEXT`(본문)을 자동으로 추출해 입력창에 채워줍니다. 직접 수정 가능.
2. **이미지 내용 직접 입력**(선택): 문의에 첨부된 이미지(SAP 오류 화면 등) 내용을 텍스트로 직접 입력하면 검색 쿼리에 포함됩니다.
3. **검색 대상 토글**: 문의(`support_cases_hyde`) / 처리내역(`support_answers_hybrid`) 중 선택.
4. **검색 결과 카드** (최대 3건, 유사도순):
   - 유사도 % — 70% 이상 초록, 50~70% 노랑, 그 외 회색
   - 접수번호 클릭 → 해당 상세 페이지로 바로 이동
   - **처리내역** 버튼 → 처리내역 팝업 열기
5. AI 서버(`192.168.10.54:8000`)가 꺼져 있거나 응답이 없으면 카드 영역에 오류 메시지가 표시됩니다.

## 설정 저장 방식

검색 조건은 브라우저 **쿠키**에 저장됩니다. 브라우저를 닫아도 유지되며 다음 접속 시 자동으로 불러옵니다.

| 쿠키명 | 내용 |
|--------|------|
| `dsh_processer` | 처리자 이름 |
| `dsh_days` | 시작일 범위 |
| `dsh_teams` | 팀 선택 (JSON 배열) |
| `dsh_progression` | 처리상태 (JSON 배열) |
| `dsh_sort_col` | 정렬 기준 컬럼 |
| `dsh_sort_dir` | 정렬 방향 |
| `dsh_auto` | 자동 설정 여부 |

핀 고정 버튼 위치는 **localStorage** (`us-pinned-btns`)에 저장됩니다.

## 문제 해결

### API 오류 / 반영일정 조회 안 됨
- `unisupport-server`(`192.168.10.54:3001`)가 실행 중인지 확인
- `node server.js` 또는 `npm start`로 서버 재시작

### AI 유사사례 검색 / 초안 생성이 안 됨
- AI 서버(`192.168.10.54:8000`)가 켜져 있는지 확인 — `docker compose ps` (support-ai 프로젝트)
- 같은 사내망(LAN)에 있는지 확인 — 외부/재택 환경에서는 해당 IP에 접근할 수 없습니다
- 초안 생성 시 502/503 오류가 뜨면 Upstage API 키(`UPSTAGE_API_KEY`) 설정 여부 확인

### 익스텐션 업데이트 후 기능이 동작하지 않음
1. `chrome://extensions` → 익스텐션 새로고침(↺)
2. 사이트 탭 **F5** 새로고침 (필수)

### FAB 버튼이 보이지 않음
- `home.uni` 또는 `welcome.uni` 페이지에서만 표시됩니다
- 다른 페이지(개별 티켓 등)에서는 표시되지 않습니다

### 반영일정 알림이 중복 발송됨
- `new-deploy` 서버와 `unisupport-server`가 동시에 실행 중인지 확인
- `unisupport-server` 하나만 실행해야 합니다

### Mixed Content 오류
- 익스텐션 내 `relay.js` + `background.js` 구조로 HTTPS → HTTP 요청을 우회 처리합니다
- 오류 발생 시 탭 새로고침 후 재시도

### 플로팅 버튼 삭제 방법
- **F8** 키를 누르면 각 플로팅 버튼에 **×** 버튼이 나타납니다
- × 클릭 시 해당 버튼이 제거되고 localStorage에서도 삭제됩니다
- 또는 FAB 메뉴를 열고 해당 항목의 핀 아이콘을 다시 클릭해도 제거됩니다

## 파일 구성

```
UniSupport-extension/
├── manifest.json          # 익스텐션 설정 (MV3)
├── background.js          # Service Worker — HTTP fetch 프록시
├── relay.js               # ISOLATED world 브릿지 (MAIN ↔ background)
├── content-search.js      # 검색 조건 자동 설정 · 그리드 정렬 로직
```
